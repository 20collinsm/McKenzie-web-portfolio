Web Accessibility Checklist
User Characteristic Accessible Design Tip
Unable to see Code all images with ALT text
Unable to percieve color Avoid using color alone to convey information
Unable to use mouse Be sure all website features can be accessed using keyboard
Unable to hear Add captions to multimedia
Prone to having seizures Avoid content that flashes
Easily distractable Keep the design simple

Accessibility Review of the Bordentown Regional High School Home Page

  • Problems
    • Unnecessary amounts of text in some areas (all of which have internal links to more information on the subject)
    • 14 links to PDF documents
    • 1 "noscript" element
  • Solutions
    • Include less alternate text and limit captions to breif summaries
    • Have alternate ways to access PDF documents in case there is an accessibility problem
    • Ensure that the "noscript" content is visible because it will only be seen by users who have JavaScript turned off
  • Conclusion

    • Considering the size of the Bordentown Regional High School website, there are very few bugs throughout the code. Overall, the website is works very well and allows the users to understand the content while still presenting it in a creative way. However, in order to maximize efficiency, the website should have smaller captions and provide alternative ways to access some key elements. This will speed up the loading time and will make it much easier for users to navigate the site.
